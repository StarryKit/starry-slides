# ADR-0007: Limit generated deck copies

- Status: accepted
- Date: 2026-05-05
- Amendment: [ADR-0012](./0012-keep-sample-slides-out-of-project-git.md)
  supersedes the `sample-slides/` Git tracking guidance in this ADR.

## Context

The editor app and e2e suite both need generated slide decks, but the repository
had accumulated overlapping copies:

- `sample-slides/`
- `.e2e-test-slides/`
- `generated/starry-slides-project-overview/`
- `.tmp/generated-deck-baseline/`

This made it unclear which deck `pnpm dev` loaded and which copy e2e reset from.
It also created unnecessary write paths when a test or manual edit saved slides.

## Decision

Keep only two generated deck copies:

1. `.e2e-test-slides/`
   - ignored by git
   - generated by `pnpm prepare:regression-deck`
   - used as the e2e working deck when Vite runs with
     `STARRY_SLIDES_DECK_SOURCE=e2e`
   - reset between e2e tests from an in-memory startup snapshot

2. `sample-slides/`
   - the App's ignored local default deck
   - loaded by normal `pnpm dev`
   - updated by `pnpm editor:e2e:generate-deck` when the e2e fixture should
     become the local App default deck
   - not tracked by this project repository's Git index

Do not create additional generated deck mirrors such as
`generated/starry-slides-project-overview/` or `.tmp/generated-deck-baseline/`.

## Consequences

Benefits:

- `pnpm dev` has a single clear source: `sample-slides/`
- e2e runs can mutate and reset their own ignored working copy without touching
  the App default deck
- updating the App default deck is explicit through one command
- browser editor edits to `sample-slides/` do not create project source diffs
- local cleanup is simpler because there is no extra `.tmp` or source mirror

Costs:

- e2e reset uses an in-memory snapshot captured when Vite starts, so
  `pnpm test:e2e:prepare` must run before the e2e server starts
- a fresh checkout may need to generate the ignored App default deck before
  running workflows that expect `sample-slides/`
- the App default deck must be intentionally refreshed when the e2e fixture
  changes

## Alternatives considered

### Keep a `.tmp` reset baseline

Rejected. It duplicated `.e2e-test-slides/` and made reset source
selection harder to reason about.

### Keep a third source mirror under `generated/starry-slides-project-overview/`

Rejected. It represented the same deck under another name and was not needed for
normal dev or e2e execution.

### Serve e2e directly from the App default deck

Rejected. E2E tests intentionally mutate slide HTML and need isolated reset
behavior. Serving the App default deck during e2e would make tests write to the
long-lived default deck unless extra copy logic was reintroduced.

## Non-goals

This decision does not:

- define a cloud or multi-deck storage model
- change the Starry Slides Contract
- require `pnpm dev` to regenerate slides on every startup
- change how agent-generated user decks are installed into the App default path

## Implementation Plan

Affected paths:

- `vite.config.ts`
- `e2e/tools/prepare-regression-deck.mjs`
- `package.json`
- `.gitignore`
- `README.md`
- `src/node/CONTEXT.md`

Steps:

1. Remove `.tmp/generated-deck-baseline/` from the Vite reset flow.
2. Remove `generated/starry-slides-project-overview/` from generation and save
   targets.
3. Make `prepare:regression-deck` generate only `.e2e-test-slides/` by
   default.
4. Make `test:e2e:serve` run Vite with `STARRY_SLIDES_DECK_SOURCE=e2e`.
5. Make `editor:e2e:generate-deck` generate `.e2e-test-slides/` and
   sync the result to `sample-slides/`.
6. Keep both `.e2e-test-slides/` and `sample-slides/` ignored by this project
   repository.
7. Update README and app context docs to describe the two-copy policy.

## Verification

- [ ] `pnpm editor:e2e:generate-deck` writes `.e2e-test-slides/` and the ignored
      local `sample-slides/`.
- [ ] `pnpm prepare:regression-deck` writes `.e2e-test-slides/` only by
      default.
- [ ] `pnpm dev` serves `sample-slides/`.
- [ ] `git ls-files sample-slides` prints no tracked files.
- [ ] `pnpm test:e2e` serves `.e2e-test-slides/` and passes.
- [ ] No code or docs reference `.tmp/generated-deck-baseline/` or
      `generated/starry-slides-project-overview/`.
