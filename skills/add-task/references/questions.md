# Required Questions

Use these prompts to collect only the missing information needed to fill the
 implementation issue template.

## Required fields

- milestone: which roadmap milestone this task belongs to
- summary: short task statement
- problem: why this task exists
- scope: what is included
- non-goals: what is explicitly excluded
- acceptance criteria: how completion will be judged
- tests: what unit, regression, or E2E coverage is required
- context links: roadmap, ADRs, and relevant package contexts

## Question order

Ask the smallest useful set of questions.

Preferred order:

1. which milestone and which implementation slice is this for
2. what problem should this issue solve
3. what is in scope and what is out of scope
4. what observable acceptance criteria should this issue meet
5. what tests are required beyond the default `pnpm verify`

## Draft review

Before creating the issue, always show:

- issue title
- full issue body
- target milestone section to update in `ROADMAP.md`

Wait for explicit confirmation before publishing.
